Overview

Tool for Spotify users, built with React, Spotify API and with backend hosted on Heroku.

Features

  • Log in with Spotify.
  • Option to display public and private playlists of user + information about them (num. of songs, name and cover art).
  • Option to customize pare down process.
  • Duplicate user playlist with new number of songs (that user choose to).

How it works

After user successfully logged in with his account, we use Spotify API endpoint to GET list of all playlists of current user - then we display information about all of them (name, num. of songs, cover art) on step 1. Here user can select which playlist he want to pare down, and customize that whole process on step 2 (num. of songs, copy recently songs or the oldest ones in playlist). Then we use Spotify API endpoint to POST new playlist with: same name as that one that user selected and with proper songs that we GOT from original playlist by using Spotify API endpoint.

Challenges

  • It was my first time using this type of an API, so it took me some time to fully adapt to using it. Especially when I was implementing option to sort current playlist (recently added or oldest ones) - choosing correct query parameters, then fetching and displaying correctly sorted tracks.

Future development

I would like to redesign the whole website and add option to pare down playlists from not only Spotify, but from other music streaming services too.
